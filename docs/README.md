# Documentation site

Static site for `@prompty-tools/mcp`. The landing, getting-started, and
concepts pages are plain HTML; the per-resource tool reference pages are
generated from the tool schemas the server registers at runtime.

## Local preview

```sh
npm run docs:serve
```

This builds the package, generates `docs/tools/*.html` from `dist/index.js`,
and starts a local HTTP server at <http://localhost:4173>.

## Deployment

A GitHub Actions workflow (`.github/workflows/docs.yml`) builds and deploys
this folder to GitHub Pages on every push to `main`.

In the repository settings, set **Pages → Source** to **GitHub Actions**
(not "Deploy from a branch") so the workflow's artifact is what gets
served.

## Layout

| Path                   | Purpose                                            |
| ---------------------- | -------------------------------------------------- |
| `index.html`           | Landing page                                       |
| `getting-started.html` | Create a key, install per client, verify           |
| `concepts.html`        | Building blocks, scopes, pagination, limits, errors|
| `tools/*.html`         | Generated by `npm run docs:build` - `.gitignore`'d |
| `assets/styles.css`    | Shared stylesheet (copied from prompty-tools-core) |

## Notes

- The tool reference pages are generated by `scripts/build-docs.mjs` from the
  same zod schemas the MCP server registers, so the documented surface cannot
  drift from the implemented one.
- Unlike prompty-tools-core's docs, examples are not runnable in the browser:
  an MCP server is a stdio process, not a browser-importable library.
- This folder is excluded from the npm publish (see `files` in
  `package.json` and `.npmignore`).
